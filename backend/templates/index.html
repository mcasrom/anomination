📋 Guía RGPD: ¿Qué datos son necesarios?

Según el Artículo 5(1)(c) del RGPD (minimización de datos), solo deben tratarse los datos estrictamente necesarios para la finalidad del trámite. Esta tabla muestra, para cada tipo de documento, qué campos son necesarios y cuáles son excesivos.

🔍 ¿Cómo detecta Anonymation el tipo de documento?

La app usa un sistema híbrido de detección con tres fases. No se envía ninguna imagen a servidores externos.

1

OCR por Tesseract OCR

El motor Tesseract escanea el texto visible en la imagen buscando palabras clave como "DNI", "PASAPORTE", "CONDUCIR", "NIE", "RESIDENCIA", "SANITARIA" o "EMPADRONAMIENTO". Cada tipo de documento tiene un diccionario de términos característicos. Se puntúa cada acierto y se selecciona el tipo con mayor coincidencia.

2

Proporción de aspecto RATIO

Cada formato de documento tiene una relación ancho/alto característica (p.ej. DNI ~0.68, pasaporte ~0.71, tarjeta sanitaria ~0.62). Si el OCR no da resultados concluyentes, la app usa la proporción como señal secundaria.

3

Reglas RGPD por tipo RGPD

Una vez identificado el tipo, se aplican las reglas de minimización del RGPD. Cada campo tiene asignada una etiqueta de necesario o excesivo según el principio de minimización (Art. 5) y la jurisprudencia de la AEPD. El usuario puede ajustar manualmente qué campos redactar.
